Как данные доходят до экрана

Общая карта: 5 страниц, под каждой — элементы и цепочка функций от базы до пикселя. Кликни элемент, где есть ссылка — откроется детальный разбор. Числа в примерах обезличены.
КАРТА

Фудкост

/foodcost ДЕТАЛЬНО ГОТОВО
build_food_cost() → Foodcost → GET /api/foodcost → FoodcostRepository → foodcost.mapper.ts
Чистый фудкост ↗
_fold/_facts → totals.cost, revenueWithCost → buildOverviewClean·fcPct() → «31,7 %»
Юниты (К/Б/В) ↗
_unit_cost_sums → units[] → buildUnits·fcPct() → бары по юнитам
Категории ↗
_group_cost_sums → groups[] → buildCategories·fcPct() → вкладки категорий
Скидки ↗
_discount_sums → discounts{5} → buildDiscounts() → 3 плашки
Потери ↗
_compliment_sums → losses → buildLosses() → строки потерь

Дашборд

/dashboard ОБЗОР
build_dashboard / _chart / _kpi → GET /api/dashboard(+/chart,/kpi) → dashboard.mapper.ts (слои)
KPI-карточки
_totals·build_dashboard_kpi → kpis.* {value, prevValue} → buildDashboardKpiLayer·lfl() → «выручка + LfL %»
График выручки по дням
_daily·build_dashboard_chart → revenueByDay[] → revenue-days-chart.utils → столбцы
Донат долей К/Б/В
_unit_sums → units[] → buildDashboardRevenueCategories → дуги доната
Мини-панель фудкоста
_unit_sums → units[] → buildDashboardFoodcostMini·fcPct → % по юнитам
Панель склада
стаб data/warehouse.json → totals{value,byStore} → buildStockFromWarehouse → остатки по складам

Продажи

/sales ОБЗОР
build_sales() → GET /api/sales → sales-aggregation.utils.ts (фронт считает rev/gp/fc/ABC)
Таблица позиций
build_sales·GROUP BY name → positions[] {qty,price,unitCost} → positions-table-organism → строки таблицы
ABC-анализ
positions[] → abc-analysis.utils (80/95%) → группы A/B/C
Донат структуры
positions[] → sum(rev) по unit → дуги доната («o» вне доната)

Склад

/warehouse В РАЗРАБОТКЕ · СТАБ
v2-контракт спроектирован (schemas/warehouse.py), эндпоинт пока отдаёт стаб — миграция после согласования
Тоталы по складам
снапшот · SUM(value) qty>0 → totals[StoreValue] → итоги К/Б/В
Таблица остатков
stock_balances дня → positions[] → таблица состава
График динамики
SUM по (day,store) → dynamics[] → линия стоимости во времени
Минусовые остатки
строки qty<0 → negativeStock → предупреждение
